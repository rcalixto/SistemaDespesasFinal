3. WORKFLOWS / LÓGICA DO SISTEMA

====================================================================

CRIAR AUTOMATICAMENTE OS WORKFLOWS:

ADIANTAMENTOS

1. Usuário cria solicitação → Status = “Solicitado”

2. Diretoria recebe notificação → Aprova/Rejeita

- Se aprovado: Status = “AprovadoDiretoria”

3. Financeiro recebe notificação → Aprova/Rejeita

- Se aprovado: Status = “AprovadoFinanceiro”

4. Pagamento é registrado → Status = “Pago”

5. Usuário envia PRESTAÇÃO → Status = “PrestacaoPendente”

6. Financeiro valida valores → Status = “Concluído”

REEMBOLSOS

1. Solicitação → “Solicitado”

2. Diretoria aprova → “AprovadoDiretoria”

3. Financeiro aprova → “AprovadoFinanceiro”

4. Pagamento registrado → “Concluído”

PASSAGENS AÉREAS

1. Solicitação

2. Aprovação diretoria

3. Aprovação financeiro

4. Emissão → registrar valores no módulo VIAGENSEXECUTADAS

HOSPEDAGENS

1. Solicitação

2. Aprovação diretoria

3. Aprovação financeiro

4. Reserva → registrar valores no módulo HOSPEDAGENSEXECUTADAS
